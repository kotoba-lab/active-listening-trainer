## v1からv2の変更概要

### 1. 構造変更

| 変更内容 | 理由 |
|----------|------|
| **D-0（書くリフレクション）を新規追加**（入門→初級ブリッジ） | A〜C（生成ゼロ）→ D（フル生成）のジャンプが認知負荷的に急すぎた。Sweller (1988) のCognitive Load Theoryに基づき、口頭不要の中間ステップを挟んだ |
| **L（助言衝動の自覚）を中級→初級へ移動** | IとLは同じ構造（衝動の言語化）であり、初級段階で並べて扱う方が自然。中級に置く理論的根拠がなかった |
| **O（感情引き込まれチェック）を上級→補助ツール（横断）へ再分類** | 全レベルで使える定常チェックであり、上級に固定する意味がなかった。特にE・N前後での使用を推奨する形に変更 |
| **D練習末尾にP簡易版を統合** | 上級まで終結を学ばない設計上の穴を修正。Dで「短い要約＋ありがとう」だけを体験させ、Pで完全版を学ぶ進行に |
| **Q（Affirmation/肯定）を初級に新規追加** | OARS の A が独立した練習として存在しなかった。感情への共感（リフレクション）と存在への承認（Affirmation）は異なるスキルであり分離が必要 |
| **R（具体化促進）を中級に新規追加** | 「もう少し具体的に」の明確化質問が設計から抜けていた。KがEvoking型の問いかけを扱うのに対し、Rは話し手の語りを精緻化するclarification型で別スキル |

### 2. 引用の修正

| 練習 | v1の問題 | v2での修正 |
|------|----------|----------|
| **A** | Lieberman (2007) は「自己感情」のaffect labeling研究。他者感情検出への適用は飛躍 | Ickes (1997) を主引用に格上げ。Liebermanは補足として注記付きで残した |
| **E** | 「EFTの実証研究」と書いて引用なし | Greenberg & Watson (2006) を追加 |
| **G** | error-based learning の引用なし | Kornell & Bjork (2008) を追加 |
| **I/L** | ACTのdiffusionを「衝動を言語化→次回の抑制率UP」の根拠に使っていたが、ACTはむしろ言語化による「制御」を目的としない立場 | Iに注記を追加。LにWells (2000) metacognitive therapy を主引用として追加。Hayes (ACT) は補足 |
| **M** | de Shazer & Berg (1992) はpost-structural理論の論文。scaling questionsの一次出典は1988が適切 | de Shazer (1988) *Clues* に差し替え。Gross (1998) cognitive reappraisal も追加 |
| **N** | 「判断力低下がburnoutより先に判断遅延として現れる」がMaslach & Leiterに直接記載されているか不明 | Maslach削除。Figley (1995) と Pearlman & Saakvitne (1995) で根拠を再構成 |
| **P** | Walter & Peller (1992) の書誌情報が不正確（原著タイトルが異なる） | 正式タイトル *Becoming Solution-Focused in Brief Therapy* に修正 |

## 次のレビュアーへ：確認をお願いしたい事項

### A. 引用の一次確認（優先度：高）

以下は「記載内容は妥当と判断したが、ページ・章レベルの確認まではしていない」引用。専門家レビューで確認してほしい。

1. **Ickes (1997)**：empathic accuracy が「意識的注意の向け方で精度が変わる」という主張が本書の主要テーゼに含まれているか確認
2. **Wells (2000)**：metacognitive awareness の訓練が衝動制御を高めるという記述が第何章にあるか
3. **Bavelas et al. (2000)**：listener responses as collaborative process — この論文の主張がR練習の「具体化要求は協働行為」という記述を直接サポートするか確認
4. **Galinsky & Moskowitz (2000)**：仮説的状況提示による感情の多面的探索について、K練習への適用が適切か

### B. 新規練習（D-0 / Q / R）の教育設計

以下は今回新規追加した練習。カリキュラム設計の観点から検討が必要。

1. **D-0の必要性**：A→D のジャンプを埋めるブリッジとして設計したが、これが「追加の複雑さ」を生んでいないか。別案として「Dの最初の数回を書き形式で行う」というバリアント記述にする方が実装しやすい可能性
2. **Qの位置**：初級の中でD〜I と並置しているが、D練習（コア）に統合（D評価軸にAffirmation観察を加える）した方がシンプルになるかもしれない。独立練習として維持する教育的理由を確認したい
3. **R（具体化促進）の難易度**：現在は中級に置いているが、「なぜ」型との区別（注意点）が初学者には難しく、上級に移した方が良い可能性がある

### C. 抜けている可能性があるスキル（未解決）

今回のレビューで指摘したが、v2では追加しなかった領域。次レビュアーが判断してほしい。

- **非言語・準言語への注意**（SOLER: Egan, 1975）：音声のみのアプリでは実装困難。対面練習として補足資料にするか、入門に1項目追加するか
- **怒りへの傾聴**：攻撃的・批判的な話し手への対応。現行シナリオに含まれているかもしれないが、練習として独立させる価値があるか
- **継続セッションの関係維持**：複数回にわたる傾聴の一貫性。「前回話したこと」の参照など。現行設計はすべて1回完結

### D. 難易度の段差（未解決）

- 推奨セットの表が刷新されたが、**初級（発展）が4練習・40分と重い**。E+G+H+Q を2回に分けるか、E単独で1セッションにする方が現実的かもしれない
- **入門セット（A→B→C）に B が含まれているが**、Bは話し手の「選択」回答が必要なため Solo 不可。Solo 想定の練習と Pair 必須の練習を推奨セット表で区別した方が良い

## アプリ実装（Layer1）との対応メモ

現行アプリ（AL-001〜AL-030、選択式/ガイド付き/自由回答 + MatchRule採点）との接続：

| 練習 | 実装可能性 | 必要な追加開発 |
|------|------------|----------------|
| B | 高（既存選択式で対応） | 3択形式への調整のみ |
| C | 高（選択式で「どれが正しいか」問題） | シナリオ追加のみ |
| D | 高（自由回答 + LLM採点で対応） | LlmScoringPromptBuilder の接続（既存） |
| D-0 | 中（書き込み→判定のみ） | 採点ロジックを「合・近・外」3段階に |
| G | 中（2ターン採点） | 「外れ判定→再入力」のターン管理UI |
| H | 中（タイマーロック） | 提出ボタンを3秒ロックする簡易実装 |
| O | 中（前後スライダー） | 練習前後の0-10スライダーUI |
| Q | 低（Affirmation採点が難しい） | LLM採点でAffirmationの有無・根拠付きかを評価 |
| R | 低（問いかけの「なぜ」判定が難しい） | LLM採点が必要 |
| E / I / J / K / L / M / N / P | 低（Pair/Trio前提 or 複数ターン対話） | アプリより対面練習として設計する方が適切 |

**実装の推奨優先順：O（3日以内）→ H（1週間以内）→ G（2ターン設計）→ D-0（採点3段階化）**
