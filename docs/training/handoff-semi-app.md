## アイデアの概要

**「アプリ外の対面練習」にLLMフィードバックを繋ぐ、コスト0の半アプリ構想。**

完全なアプリ化（リアルタイム対話評価等）は現状困難な練習（Pair/Trio形式）に対して、
「ユーザーが練習後に入力 → アプリがLLM向けプロンプトを生成 → ユーザーが貼り付けてフィードバックを得る」
という運用フローで学習品質を担保する。

---

## フロー

```
[練習実施（対面 Pair/Trio または Solo）]
        ↓
[本アプリ：ガイド表示 + 練習後の入力フォーム]
        ↓
[本アプリ：入力内容をLLM向けプロンプトに整形（エビデンス込み）]
        ↓
[ユーザーがワンタップでコピー → Claude/ChatGPT等に貼り付け]
        ↓
[LLMが練習へのフィードバックを返す]
```

---

## プロンプト出力の構造（確定）

アプリが生成するプロンプトは以下のセクションで構成する：

```
## 練習情報
練習名: E. 深層感情掘り
目的: 表面感情の下にある一次感情を識別する

## 理論的根拠（参照情報）
- Greenberg (2002): 二次感情の下には一次感情がある。一次感情への
  アクセスが共感の深さを決定する。
- Greenberg & Watson (2006): 一次感情への応答が治療関係・会話継続
  に有意な効果を示した。

## 評価基準
- 深層感情への到達度
- 表面/深層の区別が応答に反映されているか

## 今回の練習記録
[チェック] 深層感情への到達：△ 惜しかった
[チェック] 区別の明確さ：✕ 区別できていなかった
[自由記述] 怒りは特定できたが、その下への言及が遅れた。
           話し手からは「悲しみの方が近かった」とフィードバックあり。

## 依頼
上記をEFTの枠組みで評価し、次回の具体的な改善点を教えてください。
```

**エビデンスを埋め込む理由**：Claude/ChatGPT以外の検索しないLLMでも、
参照情報の揺らぎなく一定品質のフィードバックが得られるようにするため。
これは「RAGのローカル版」的発想（サーバー検索なしで文献を梱包して渡す）。

---

## 入力フォームの方針（C案）

定型項目（選択・チェックボックス）＋自由記述の組み合わせ。

各練習の評価軸を選択肢化：
- 評価軸ごとに「◎できた / ○おおむねできた / △惜しかった / ✕できなかった」
- 最後に自由記述欄（何が起きたか・話し手のフィードバック・気になった点）

---

## 実装するべき練習の優先順

フルアプリ化が困難でこの半アプリが特に有効な練習（`handoff-to-next-reviewer.md` 実装可能性「低」の練習を優先）：

| 優先 | 練習 | 理由 |
|------|------|------|
| 高 | E（深層感情掘り） | Trio形式・LLM評価が価値高い |
| 高 | I（自己開示ブロック） | 事後メモ形式なので入力親和性高い |
| 高 | L（助言衝動の自覚） | 既にSolo事後メモ設計 → そのまま入力フォーム化できる |
| 中 | G（修正応答） | 2ターンの記録が必要 |
| 中 | J（フォローアップ→要約） | Trio形式 |
| 低 | N（適応限界の判断） | 上級・デリケートな内容 |
